HBPartialRegistry Class Reference

Inherits from NSObject
Declared in HBPartialRegistry.h

Overview

HBPartialRegistry instances are objects holding a set of partials available to templates. They are the actual objects used by [HBExecutionContext] instances to store partials.

You generally do not use a registry directly but use the corresponding shorcut methods on [HBExecutionContext]

Partial registries implement the objective-C keyed subscripting API though, meaning that you can read and write partials using subscripting operators. For instance :

HBExecutionContext e = [[HBExecutionContext] new] autorelease]; HBPartial p = …; e.partials[@“my_partial”] = p;
